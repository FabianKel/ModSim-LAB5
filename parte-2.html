  1. En el modelo de Watts-Strogatz, el parmetro de recableado p es la clave para sintonizar la red entre orden y aleatoriedad. Describa qu sucede con el coeficiente de agrupamiento promedio y la longitud de camino promedio de la red a medida que p vara desde 0 hasta 1. Por qu un valor de p muy pequeo (ej. 0.01) es tan efectivo para reducir drsticamente la longitud del camino promedio sin destruir el agrupamiento local?
  • Cuando $p$ vara de 0 a 1:
    • $(p = 0)$ (anillo regular):
      • Alto coeficiente de agrupamiento (clustering local fuerte).
      • Larga longitud de camino promedio (camino tpico entre dos nodos es grande).
    • $0 < p \ll 1$ (pequeo recableado):
      • Coeficiente de agrupamiento se mantiene alto o disminuye muy poco.
      • Longitud de camino promedio cae drsticamente (se introducen atajos).
      • Resultado: mundo pequeo combinacin de clustering y caminos cortos.
    • $p \approx 1$ (red aleatoria):
      • Bajo clustering.
      • Longitud de camino promedio muy corto (propio de redes aleatorias).

Un $p$ muy pequeo reduce fuertemente la distancia sin destruir el clustering porque:

  • Un pequeo nmero de aristas "recableadas" crea atajos que conectan regiones lejanas del anillo regular.
  • Esos atajos acortan enormemente las rutas tpicas porque permiten saltar grandes tramos localmente.
  • Como solo unas pocas aristas cambian, la mayora de la estructura local permanece intacta, por eso el clustering se conserva mientras la distancia promedio se reduce mucho.
  • Intuitivo: unas pocas autopistas entre barrios reducen tiempos de viaje sin eliminar las calles locales.
  1. Piense en la red de amistades dentro de una red social como Facebook o Instagram. Esta red muestra caractersticas de varios modelos: tiene comunidades muy unidas (grupos de la universidad, compaeros de trabajo), lo que sugiere un modelo de mundo pequeo. Al mismo tiempo, tiene "influencers" o celebridades con millones de seguidores, lo que es caracterstico de una red libre de escala. Cmo podra un modelo de simulacin capturar ambas propiedades? Proponga una regla generativa conceptual (no necesita escribir

cdigo) para construir una red que sea a la vez comunitaria y tenga hubs

  • Para construir redes que tengan comunidades muy cohesionadas y, a la vez, hubs con gran grado, como en el caso de influencers, conectando muchas comunidades, se puede seguir la siguiente estructura:

  1. Generar comunidades locales (bloques densos)

    • Partir la poblacin en $k$ comunidades, como universidades, empresas, condominios.
    • Dentro de cada comunidad, conectar nodos con probabilidad alta, Creando un alto clustering local.

  2. Introducir unos pocos nodos con potencial de hub

    • Seleccionar aleatoriamente o por distribucin un conjunto pequeo de nodos candidatos a influencers.
    • Para cada nuevo nodo influencer, conectar preferencialmente hacia nodos ya con alto grado global pero permitiendo conexiones hacia mltiples comunidades con un puente inter-bloques.

  3. Aadir enlaces inter-comunidad

    • Con baja probabilidad crear enlaces aleatorios entre nodos de distintas comunidades, creando atajos al estilo mundo pequeo.
    • Adems, forzar que los influencers tengan una probabilidad elevada de conectar a nodos en todas las comunidades generando hubs globales.

  4. Iteracin y balance

    • Repetir la adicin de nodos y enlaces hasta alcanzar tamao deseado. Ajustar las probabilidades para controlar la densidad interna, nmero de puentes y presencia de hub.

Esta estructura captura ambas propiedades por:

  • Los poblados densos dentro de bloques generan el clustering/comunidades observado.
  • Las conexiones preferenciales y la regla que permite a ciertos nodos (influencers) conectarse a muchas comunidades generan hubs globales que replican la cola pesada de grados.
  • La pequea probabilidad de crear enlaces aleatorios entre distintas comunidades y la presencia selectiva de hubs mantiene caminos cortos entre comunidades (caracterstica de mundo pequeo).

# Promedio y bandas de error para el tamao del componente gigante
import numpy as np
import networkx as nx
import matplotlib.pyplot as plt

# Parmetros de muestreo
N = 100                     # nmero de realizaciones por p
ps_fine = np.linspace(0.0, 0.02, 101)  # malla fina alrededor de 1/n (n=200)
rng = np.random.default_rng(12345)
n = 200

means = []
stds = []

for p in ps_fine:
    vals = []
    for i in range(N):
        G = nx.erdos_renyi_graph(n=n, p=p, seed=int(rng.integers(0,1e9)))
        comps = nx.connected_components(G)
        giant = max(comps, key=len, default=set())
        vals.append(len(giant)/n)
    means.append(np.mean(vals))
    stds.append(np.std(vals))

means = np.array(means)
stds = np.array(stds)
pc = 1 / n

plt.figure(figsize=(8,4.5))
plt.plot(ps_fine, means, label=f'Media (N={N})', color='C0')
plt.fill_between(ps_fine, means-stds, means+stds, color='C0', alpha=0.3, label='1 ')
plt.axvline(pc, linestyle='--', color='red', label=f'p = 1/n  {pc:.3f}')
plt.xlabel('Probabilidad de conexin p')
plt.ylabel('Tamao normalizado del componente ms grande')
plt.title(f'Media   del componente gigante (n={n}, N={N})')
plt.grid(True, alpha=0.3)
plt.legend()
plt.show()
